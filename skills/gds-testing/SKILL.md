# GDS Testing

## Overview

This skill is for game testing work, not generic web or backend test execution. It helps choose the right testing approach for gameplay, systems, and performance concerns.

If the task is specifically XiaKeXing runtime automation or xiakexing_ai MCP execution, use `ai-player` instead of treating this skill as the executor.

## Mode Selection

Use **design mode** for test strategy, scenarios, and coverage planning.

Use **automation mode** for unit, integration, or end-to-end test automation ideas.

Use **performance mode** for frame rate, loading, memory, or responsiveness concerns.

Use **playtest mode** for user-facing playtest structure and feedback collection.

Use **review mode** for evaluating the quality or completeness of an existing game test effort.

## Output Format

```markdown
## Game Testing Recommendation

**Mode:**
**Focus area:**
**Recommended approach:**
**Key risks to cover:**
**Next validation step:**
```

## Why the Modes Matter

Gameplay validation, automation, and performance work answer different questions. Picking the mode first keeps the advice concrete instead of turning into a vague testing checklist.
